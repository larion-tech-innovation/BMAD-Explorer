§ PRINCIPLESFacilitation Approach

Pulled directly from bmad-brainstorming's own "Framing" guidance — verbatim, not paraphrased. These hold for the whole session, regardless of which of the 3 stances is chosen.

Aim past 100 ideas
Resist concluding — the urge to organize or wrap early is the enemy of divergence. When in doubt, push for one more idea.
Keep shifting the creative domain
Every 5-10 turns (or ~10 ideas), usually by moving to the next technique — prevents anchoring on the first few ideas.
One prompt per message
No stacked questions, no multiple-choice menus that invite lazy picking — both pull the user out of generating.
Append-only memory
Every idea, decision, and bit of direction is logged in time order and never edited or reordered — the memlog is the single source of truth for the session.

§ ACTIVATIONWhat Happens When You Talk to Carson

Every one of the 6 CIS agents runs through the exact same 8-step sequence on activation — only the persona and owned skill change. Here it is in plain language, with the technical mechanism noted alongside each step.

1
Load any custom settings
If this project has customized Carson (a different tone, extra instructions), that customization is merged in now — project-wide defaults, then team settings, then your own personal tweaks. customize.toml base → team → user, merged via resolve_customization.py.
2
Run any setup steps configured for this project
Optional project-specific steps that need to happen before Carson greets you.
3
Become Carson
Fully adopts the persona — tone, personality, communication style — and stays in character for the whole conversation, even when you switch topics.
4
Remember anything set up to persist
Loads any standing facts/context this project wants Carson to always know. If a referenced file is missing, it's silently skipped — never invented.
5
Read your project settings
Your name, your preferred language, and where output files should be saved — read from this project's config.
6
Say hello
Greets you by name, in your language, with Carson's icon (🧠) so you always know who's talking. Mentions you can ask for help (bmad-help) any time.
7
Run any wrap-up setup steps
Any final project-specific steps configured to run right after greeting.
8
Get straight to it, or show options
If you already said what you want ("hey Carson, let's brainstorm"), he skips the menu and starts. Otherwise he shows a short numbered list of what he can do and waits for you to pick.

§ SKILL OWNEDbmad-brainstorming

Brainstorming core module skill
36 techniques · 7 categories · no fixed step count (mode-routed)
This is the one skill in the whole CIS module that isn't CIS's own — it belongs to BMAD's shared core, and Carson is simply the agent CIS uses to reach it. Three stances to choose from: Facilitator (you do the generating), Creative Partner (Carson joins in), or Ideate for me (Carson runs the whole thing). No fixed number of steps, because the session's length and shape depend entirely on which stance you pick and how far you want to push.
