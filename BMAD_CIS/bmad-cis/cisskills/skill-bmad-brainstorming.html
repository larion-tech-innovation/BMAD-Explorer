§ WHATBrainstorming — What does this skill do?

This skill runs a full brainstorming session: someone brings a topic and wants to generate far more, and far better, ideas than they'd reach alone — pushing past the obvious with sharper questions and harder constraints, with no rush to finish. It draws from a library of 36 techniques across 7 categories (Collaborative, Structured, Creative, Deep, Theatrical, Wild, Introspective).

Three Stances

Before technique selection, the user picks a stance — it governs how much of the ideation work Carson does versus draws out of the user.

  • Facilitator — Carson never supplies ideas; a forcing function for the user's own thinking.
  • Creative Partner — Carson facilitates and plays along, trading ideas back and forth; each idea's authorship (user vs. coach) is tagged.
  • Ideate for me — Carson runs the whole session and shows the user the result.

The chosen stance holds for the whole session and does not change which techniques are available.

§ WHYDesign Rationale

Why this skill has no fixed step count

Unlike CIS's own 4 workflow skills, brainstorming has no <step n> sequence — it's intent/mode-routed, not linear.

The session's shape depends entirely on which of the three stances the user picks and how long the ideation runs — the point is to "aim past 100 ideas and resist concluding," which is fundamentally open-ended, not a fixed procedure with a predetermined number of stages. Forcing a step count onto it would misrepresent how the skill actually behaves.

Why this is a core skill, not a CIS skill

Brainstorming is useful far beyond creative work — BMM and other BMAD modules use the exact same skill. Rather than duplicate it inside CIS, the module cross-lists it into Carson's menu so CIS users get the same well-maintained brainstorming capability everyone else does, while its ownership and maintenance stay centralized in core.

§ HOWUsing This Skill

Before calling this skill

  • Have your topic and the goal or "why" behind it ready — the opening question asks for both together.
  • Decide (or let Carson ask) which stance fits: Facilitator, Creative Partner, or Ideate for me.
  • If resuming a prior session, Carson checks the output folder on activation and offers to continue.

Guardrails

Grounded in the core SKILL.md's own "Framing" section, verified this run.

  • Aim past 100 ideas; resist concluding — push for one more when in doubt.
  • Keep shifting the creative domain every 5-10 turns to prevent anchoring.
  • One prompt per message while in dialogue — no stacked questions, no multiple-choice menus that invite lazy picking.
  • Makes it "safe for wild ideas to surface" while staying "precise about which ones rise" — psychological safety, "yes-and" improvisation.
  • Every idea logged to an append-only session log — never edited in place, only appended.
