§ WHAT Teach Me Testing — What does this Skill do?

overview Overview & Architecture

Skill code
bmad-teach-me-testing
Menu code
TMT
Goal
Deliver a self-paced, multi-session curriculum that teaches testing from fundamentals to advanced practices — scalable across an entire team without consuming senior engineer time.
Murat's Role
Master Test Architect and Teaching Guide — this is a collaborative learning partnership, not a one-way lecture. Murat brings TEA expertise; the learner brings real project context.
Architecture
Continuable sessions with state persistence — pause and resume at any time. Uses step-file architecture: micro-files, just-in-time loading, mandatory sequential enforcement.
Knowledge base
51 TEA knowledge fragments · 9 workflow references · content adapts to role (QA / Dev / Lead / VP)
Output
Session notes (×7) · Progress file (YAML) · TEA Academy Completion Certificate

TEA Academy turns deep testing expertise into something every team member can access — no waiting for a senior to teach, no need to be in the same room.

§ WHY Why TEA Academy?

vs TMT vs Ad-hoc Self-learning

No senior engineer needed to teach directly. A new team member can reach the team's testing standard in 1–2 weeks without affecting anyone else's capacity.

ContinuableMust remember everything after a session — can't resume where you stoppedPause & resume at any time — progress file preserves full state

assessment Assessment System quiz · 70% pass

Each session (S1–S6) ends with a 3-question multiple-choice quiz. Session 7 is completion-based (no quiz). Passing threshold is ≥ 70% (2 of 3 correct). On failure, the learner can review content or continue — the score is always recorded.

§ HOW How to Activate & Execute

activation Activation & 3 Operation Modes

Trigger phrases

  • "lets learn testing" — start or continue a learning session
  • "I want to study test practices"
  • Select [TMT] from the TEA agent menu (after "talk to Murat")

3 Operation Modes

  • Create · C Start a new session or resume existing progress. Trigger: "learn", "start", "resume", "continue". Entry point: steps-c/step-01-init.md
  • Validate · V Review workflow quality and generate a validation report. Trigger: "validate", "review", --validate. Entry: steps-v/step-v-01-validate.md
  • Edit · E Modify workflow content or session structure. Trigger: "edit", "modify", --edit. Entry: steps-e/step-e-01-assess-workflow.md

workflow-steps Workflow Steps 3 branches

Uses step-file architecture — each step is a standalone instruction file loaded only when its turn arrives (just-in-time). Steps are organized into 3 branches by operation mode. Execution order is mandatory — no skipping or reordering.

Branch · steps-c · Create mode
  • step-01-initGreet the learner, detect existing progress from prior sessions, route to continuation (progress found) or fresh assessment (first run).
  • step-01b-continueResume the workflow from a prior session — load the progress file, display the session dashboard with completion status, route to the session menu hub.
  • step-02-assessCollect role (QA / Dev / Lead / VP), experience level, learning goals, and pain points to customize examples and session order throughout the curriculum.
  • step-03-session-menuCentral hub — displays all 7 sessions with completion status and scores, allows non-sequential session selection, routes to a session or the completion ceremony. All sessions return here after completing.
  • step-04-session-01Session 1 · Quick Start (30 min) — TEA Lite intro, Automate workflow walkthrough, 3-question quiz, generates session notes, updates progress file.
  • step-04-session-02Session 2 · Core Concepts (45 min) — Testing as engineering philosophy, P0–P3 risk matrix, Probability × Impact scoring, Definition of Done (7 principles).
  • step-04-session-03Session 3 · Architecture & Patterns (60 min) — Fixture composition, network-first pattern, data factories, step-file architecture.
  • step-04-session-04Session 4 · Test Design (60 min) — Test Design workflow, risk and testability assessment, coverage planning by test levels.
  • step-04-session-05Session 5 · ATDD & Automate (60 min) — ATDD red-green TDD cycle, Automate workflow for existing code, API testing without browser.
  • step-04-session-06Session 6 · Quality & Trace (45 min) — Test Review 5 dimensions, quality scoring 0–100, Trace workflow, release gate decisions.
  • step-04-session-07Session 7 · Advanced Patterns (ongoing) — Menu-driven exploration of 42 TEA knowledge fragments across 6 categories. No quiz; completion-based (score = 100).
  • step-05-completionGenerate the TEA Academy completion certificate with session scores, skills checklist, and artifact paths. Requires all 7 sessions completed with average ≥ 70%.
Branch · steps-e · Edit mode
  • step-e-01-assessIdentify what needs editing in the workflow — gather requirements, scope modifications, assess impact on existing session content or step sequence.
  • step-e-02-applyApply approved edits to workflow files while maintaining step-file integrity, sequential enforcement rules, and BMAD quality standards.
Branch · steps-v · Validate mode
  • step-v-01-validateValidate the workflow against BMAD quality standards — check step sequencing, content accuracy, quiz correctness, progress tracking logic — and generate a validation report.

artifacts Progress Tracking & Output Artifacts

Progress is automatically saved after each session. The workflow detects existing progress on restart and resumes where the learner left off — no data is lost on interruption.

Output Artifacts
Progress file{test_artifacts}/teaching-progress/{user_name}-tea-progress.yaml
Session notes × 7{test_artifacts}/tea-academy/{user_name}/session-{N}-notes.md
Completion certificate{test_artifacts}/tea-academy/{user_name}/tea-completion-certificate.md

Progress file tracks

  • Per-session status: not-started / in-progress / completed
  • Per-session score (0–100) and completion date
  • Overall completion percentage (sessions_completed / 7) × 100
  • Learner role and experience level (set on first run)
  • Next recommended session
  • stepsCompleted array — full audit trail of executed steps

Completion certificate includes

  • Session completion dates and quiz scores per session
  • Skills acquired checklist — aligned to TEA methodology
  • Learning artifact paths (notes files)
  • Recommended next steps after Academy completion

Completion requirements

  • SessionsAll 7 sessions must be completed (minimum)
  • Passing score≥ 70% per session quiz
  • Average score≥ 70% across all sessions to receive certificate
  • Skip penaltySkipping S1 (Intermediate/Experienced) forfeits certificate eligibility

extend Extending the Curriculum — Adding a New Session 8 steps

The TMT curriculum is fully modular. Each session is defined across four data files plus one step file — you can add a new session without touching any existing session logic.

The example below adds Session 8: Contract Testing as an optional advanced session. Adapt the values to your topic. All file paths are relative to .claude/skills/bmad-teach-me-testing/.

Step 1 · Plan the session content

Before touching any file, define the following. Having these answers ready makes all subsequent steps straightforward.

  • Session IDKebab-case identifier used as key across all data files. Convention: session-NN-topic-name. Example: session-08-contract-testing
  • NameDisplay name shown in the session menu. Example: "Contract Testing"
  • DurationEstimated learner time. Match one of the existing patterns: "30 min", "45 min", "60 min", or "ongoing"
  • DifficultyOne of: beginner, intermediate, advanced
  • ObjectiveOne sentence — what the learner can do after this session. Example: "Learn consumer-driven contract testing with Pact.js"
  • Knowledge fragmentsWhich .md files in resources/knowledge/ cover this topic? List 2–4 files. If none exist, create them first.
  • Quiz questionsDraft 3 multiple-choice questions (A/B/C/D) with correct answers and explanations. Write these before creating the step file — they drive the teaching structure.
  • PrerequisitesWhich sessions should be completed first? Example: session-05-atdd-automate
  • Target rolesWhich roles benefit most? QA / Dev / Lead / VP (can be all four)
  • Mandatory or optional?See the decision table above

Step 2 · Register in data/curriculum.yaml

Add a new entry to the sessions: array. Place it after the last existing session. Also update learning_paths for each experience level that should include the new session.

# ADD to the sessions: array (after session-07-advanced)
  - id: session-08-contract-testing
    name: "Contract Testing"
    duration: "60 min"
    difficulty: advanced
    objective: "Learn consumer-driven contract testing with Pact.js"
    description: "Pact.js setup, consumer contracts, provider verification, CI integration"
    recommended_for:
      - experienced
    prerequisites:
      - session-05-atdd-automate

# UPDATE learning_paths — add to sequences that should include this session
  experienced:
    recommended_sequence:
      - session-02-concepts
      - ...
      - session-07-advanced
      - session-08-contract-testing   # ← add here

# If MANDATORY — also update:
# completion:
#   minimum_sessions: 8   # was 7

Step 3 · Map content in data/session-content-map.yaml

Add a new top-level key matching the session ID. List the knowledge fragments the agent will load just-in-time during teaching, plus the key concepts to cover.

session-08-contract-testing:
  knowledge_fragments:
    - path: "/src/agents/bmad-tea/resources/knowledge/contract-testing.md"
      title: "Contract Testing with Pact.js"
    - path: "...pactjs-utils-overview.md"
      title: "Pact.js Utils Overview"
  key_concepts:
    - "Consumer-driven contract testing (CDCT)"
    - "Pact.js consumer test generation"
    - "Provider verification setup"
    - "CI integration with can-i-deploy"

Step 4 · Write quiz questions in data/quiz-questions.yaml

Add exactly 3 questions for the new session. Each question needs 4 options (A/B/C/D), one correct answer, and an explanation shown when the learner answers incorrectly.

session-08-contract-testing:
  passing_score: 70
  questions:
    - id: q1-cdct-benefit
      question: "What is the main benefit of contract testing over traditional integration testing?"
      options:
        A: "It is faster to write"
        B: "It decouples consumer and provider — each verifies independently"
        C: "It replaces all other test types"
        D: "It only works for REST APIs"
      correct: B
      explanation: "Contract testing decouples services: consumers define expectations (pact files), providers verify independently."
  • Questions should test understanding, not memory — focus on why and when, not syntax
  • Wrong-answer explanations are shown immediately on an incorrect response — make them instructive
  • Score is calculated as (correct / 3) × 100 — 2 correct = 67% (fail), 3 correct = 100% (pass)

Step 5 · Update data/role-paths.yaml

Add the new session to recommended_sessions and teaching_adaptations for each role that benefits from it. You do not need to add it to every role — only where it is genuinely relevant.

  qa:
    recommended_sessions:
      - ...
      - session-08-contract-testing   # ← add here
    teaching_adaptations:
      session-08-contract-testing: "Focus on consumer test generation and CI can-i-deploy gate"

  dev:
    teaching_adaptations:
      session-08-contract-testing: "Connect to TDD — write contract tests alongside feature code"

Step 6 · Create steps-c/step-04-session-08.md

This is the most important file — the agent's execution script for the new session. Copy an existing session file (e.g. step-04-session-02.md) and replace the content. The structure must follow the mandatory sequence exactly.

---
name: 'step-04-session-08'
progressFile: '{test_artifacts}/teaching-progress/{user_name}-tea-progress.yaml'
sessionNotesFile: '{test_artifacts}/tea-academy/{user_name}/session-08-notes.md'
nextStepFile: '{skill-root}/steps-c/step-03-session-menu.md'
---

# Step 4: Session 8 - Contract Testing

### 2. Update Progress File (Session Started)
### 3–5. Teaching: [concepts, load knowledge fragments one at a time]
### 6. Quiz: Validate Understanding

  • 1. Start TMT"talk to Murat" → select TMT → Create mode. In the session menu, verify Session 8 appears with the correct name, duration, and difficulty.
  • 2. Enter sessionSelect Session 8. Verify the welcome message shows the correct objective. Verify the teaching content loads the right knowledge fragments (not all at once).
  • 3. Complete quizAnswer all 3 questions — test both pass (correct) and fail (incorrect) paths. Verify score: 2 correct = 67% (below 70%, should offer review or continue).
  • 4. Check artifactsVerify session-08-notes.md is created at the correct path. Verify the progress file shows session-08-contract-testing: completed with score and date.
  • 5. Resume testExit and re-enter TMT. Verify Session 8 shows as completed in the dashboard. Verify stepsCompleted array contains 'step-04-session-08'.
