v6.10.0

Released 2026-07-03 View on GitHub ↗

Dev Auto joins the roster as an opt-in, single-pass autonomous worker; eight shared skills get sharper prompts and quieter bugs; and bmad-investigate is retired for good.

Stats at a glance — v6.9.0 → v6.10.0 Agents06 Skills29 Phases04 Workflows24→23 Steps158+→157+
NEWbmad-dev-auto

Not available→new single-pass autonomous worker for Amelia's 4-implementation phase: clarifies intent, implements, reviews, and finalizes unattended.

UPDATEDbmad-party-mode

No anti-consensus lineup→built-in club (Wildcard, Level, Killjoy, Splinter) via --party=anti-concensus-club, plus an opt-in --non-interactive flag.

UPDATEDbmad-advanced-elicitation

No Subtraction or Map Is Not the Territory→both added to the method catalog.

UPDATEDbmad-review-edge-case-hunter

Missed partial-handling gaps across fixed sets (enum, status code, flag)→now catches them via a named-set generalization pass.

UPDATEDbmad-tea

9 skills, no release-gate shortcut→adds a GATE intent routing test-review → NFR audit → trace/gate.

UPDATEDbmad-party-mode

Sessions closed after one request, standing agents dropped between turns→stays open until an explicit signal; standing agents resume instead of respawning.

UPDATEDbmad-party-mode

Docs implied Agent Teams broadcast to every member→Agent Teams relay point-to-point (mailbox-style); the lead explicitly relays turns to idle members.

UPDATEDbmad-review-edge-case-hunter

Deletion-contract audit ran as its own standalone layer→folded into this skill's own turn (the standalone pass had near-zero yield).

UPDATEDbmad-code-review

Rated diff hunks in isolation, tie-breaker inflated severity→reads surrounding code first, tie-breaker removed, deletion check delegated to Edge Case Hunter.

UPDATEDbmad-quick-dev

Prompt wording predated the Dev Auto hardening pass→backports the same clarity and robustness fixes.

UPDATEDbmad-help

Read config.yaml directly; settings could silently miss→now reads through the shared four-layer config resolver.

UPDATEDbmad-forge-idea

Original launch copy→reworded for clarity, no behavior change.

UPDATEDbmad-create-prd / bmad-edit-prd / bmad-validate-prd / bmad-create-architecture

No trigger-phrase exemption for these compatibility shims→exempted from the trigger-phrase check, so users are steered to bmad-prd / bmad-architecture instead.

REMOVEDbmad-investigate

Had its own card, detail page, and inline references→retired; reached the same conclusions as plain investigation, at higher cost.
